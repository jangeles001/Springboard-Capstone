Purpose Statement:
A fitness and diet management web app that provides personalized reccomendations and stores user data in a self-hosted database. A user will be able 
to enter their login credentials to access their personalized diet and fitness routines. I intend to make an evenly focused full-stack web application
and I will implement full CRUD functionality across all core user activities, giving users control over their fitness and diet journey.

Problem:
Many workout tracking tools require users to adopt rigid plans without personalization.
Users need a customizable platform that tailors workouts based on their own health metrics and physical needs.

Inputs/Data:
User inputed health data. (e.g., weight, age, height, lifestyle, weight goals)

User base: Health-conscious individuals of any fitness level whom want to manage their workouts-
and diet in one personalized, data-driven platform.

External API: 
wger Workout Manager: https://wger.de/api/v2/
-Exercise databases.
-Muscle group info.
-Pre-made workout templates.

USDA: https://api.nal.usda.gov
- ingredients for meal builder/logger
- All macros information

I will use this data with personalization engine to generate tailored recommendations for users.

Tech Stack:
1.) Authentication:  Crypto + JWT    Difficulty: Medium
BCrypt + Json WebToken will provide the tools I will need to authenticate users. I will pass a token to
the client stored as a cookie and will verify the token before using the payload to identify the user.

2.) Frontend: React.js          Difficulty: Medium
React efficiently builds modular, scalable interfaces. My app will benefit from:
-Reusable Components for tracking, data forms, and recommendation displays.
-Hooks, Custom Hooks, and Context API to manage UI state.
-Flexibility for integrating charts (e.g., using Chart.js for tracking and displaying various user metrics).

State Management: Zustand
Lightweight and lower learning curve than redux. A better fit for the size of my project.

Routing: Tanstack Router
File-based routing and integrated data fetching and caching. Seems to be easier to work with than React Router.

UI: Tailwind
Easy to use for inline styling of elements, great utilities, and greater control of design customization.

3.) Backend: Node.js + Express.js       Difficulty: Medium/Hard
Node.js handles asynchronous operations, which will help with fetching data from the external API and managing the personalized recommendations.
Express.js is lightweight and can manage user authentication, health data, and logged workouts. AI component uses Google gemini-2.5-flash to receive a
structured prompt with user data and provide recommendations.

Middleware 
CORS:CORS middleware is used to control and secure cross-origin HTTP requests between the client and server. It defines which domains are permitted 
to access backend resources and which HTTP methods and headers are allowed. This is especially important for protecting sensitive endpoints 
in a full-stack application where the frontend and backend are deployed on different origins.

Zod: Enforce strict data schemas for incoming requests. It ensures that payloads such as authentication credentials, form submissions, 
and API inputs conform to expected types and formats before reaching business logic. This improves application reliability, prevents 
malformed data from entering the system, and provides consistent, structured error responses.

Recaptcha: Protect public-facing endpoints‚Äîsuch as registration, login, and password recovery‚Äîfrom automated abuse

Security:
- Crypto + JWT to handle secure sign-in. 
- HTTPS for secure data transmission.
- JWT handles both authentication and user sessions. Each client is provided a jwt refresh and auth token. Once
the short lived auth token expires, the user is redirected to the refresh endpoint automatically to be assigned a new auth
and refresh token. If the refresh token expires or doesn't exits, the user is redirected to the log in page.
- MongoDB access limited via JWT + token payload. If the user uuid or public Id is not present in the token, Authorization errors are thrown
by the back end and returned to the user.
